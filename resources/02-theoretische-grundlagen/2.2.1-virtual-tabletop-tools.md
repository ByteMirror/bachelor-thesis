# 2.2.1 Virtual Tabletop Tools (VTTs)

## Kontext und Planung

### Kernthema
Definition, Funktionsumfang und technische Anforderungen von Virtual Tabletop Tools als digitale Spielplattformen für Pen-and-Paper-Rollenspiele.

### Ziel des Abschnitts
Grundlegendes Verständnis von VTTs schaffen, Anforderungen an Performance und Interaktivität herausarbeiten.

## Inhaltliche Struktur

### Definition und Zweck (1-2 Absätze)
- Digitale Simulation eines Spieltisches
- Remote-Spielmöglichkeit
- Automatisierung von Spielmechaniken
- Integration von Multimedia-Elementen

### Historische Entwicklung (1 Absatz)
- Early Web-Tools (2000er)
- Flash-basierte Lösungen
- Moderne Web-Applications
- COVID-19 als Katalysator

### Kernfunktionalitäten (2-3 Absätze)
**Basis-Features:**
- **Map/Grid System**: Spielfeld-Darstellung
- **Token Management**: Charaktere und NPCs
- **Fog of War**: Sichtbarkeitssteuerung
- **Dice Rolling**: Würfelsimulation
- **Chat/Communication**: Text/Voice/Video

**Erweiterte Features:**
- **Dynamic Lighting**: Licht und Schatten
- **Character Sheets**: Regelwerk-Integration
- **Asset Libraries**: Grafiken und Sounds
- **Automation**: Makros und Scripting
- **3D Support**: Isometrische/3D-Ansichten

### Technische Anforderungen (2-3 Absätze)
**Performance-Anforderungen:**
- Echtzeit-Interaktivität (<100ms Latenz)
- Smooth Scrolling/Zooming (30+ FPS)
- Große Maps (>10000x10000px)
- Viele Objekte (100+ Token)

**Rendering-Herausforderungen:**
- Grid-Overlay (Square/Hex)
- Layer-Management (Background/Token/Effects)
- Transparenz und Blending
- Particle Effects
- Line-of-Sight Berechnungen

### Marktübersicht (2-3 Absätze)
**Major Players:**
1. **Roll20**
   - Web-basiert, Browser-only
   - Canvas-Rendering
   - Größte Nutzerbasis

2. **Foundry VTT**
   - Self-hosted, Node.js
   - WebGL-Rendering
   - Modulares System

3. **Fantasy Grounds**
   - Desktop-Application
   - Native Rendering
   - Regelwerk-Lizenzen

**Technologie-Stack Vergleich:**
| VTT | Rendering | Platform | Performance |
|-----|-----------|----------|-------------|
| Roll20 | Canvas 2D | Web | Mittel |
| Foundry | WebGL | Web/Desktop | Hoch |
| Fantasy Grounds | Native | Desktop | Hoch |
| Tabletop Simulator | Unity 3D | Desktop | Sehr Hoch |

### Performance-Kritische Aspekte (1-2 Absätze)
**Bottlenecks:**
- Asset Loading (Textures, Sounds)
- Viewport Rendering bei Zoom/Pan
- Token Animation und Bewegung
- Fog of War Calculations
- Network Synchronization

**Optimierungsansätze:**
- Level-of-Detail (LOD)
- Viewport Culling
- Texture Atlasing
- Object Pooling
- Delta Synchronization

## Spezifische Herausforderungen

### Browser vs. Desktop
- Web: Sandbox-Limitierungen
- Desktop: Installation/Updates
- Hybrid: Electron-Apps

### Single vs. Multiplayer
- State Synchronization
- Konfliktauflösung
- Latenz-Kompensation
- Authoritative Server

### 2D vs. 3D
- Performance-Unterschiede
- Komplexität der Implementierung
- User Experience
